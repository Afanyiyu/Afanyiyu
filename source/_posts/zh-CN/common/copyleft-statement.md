我是 Copyleft 人，但本页的声明只适用于我自己在业余时间开发的产品。这里的内容不适用于「并非我个人的项目」（如合作项目等）和给别人做的私有项目。此外，本博客不属于 Copyleft 范围，但本博客仍然 [开源](https://github.com/Afanyiyu/Afanyiyu)。

# Copyleft

gnu.org 上有关于 [Copyleft 的详细说明](http://www.gnu.org/licenses/copyleft.zh-cn.html)。

# 支持

基础 Copyleft 的做法如下：

- 免费

- 开源

- 软件属于公共领域

- 软件可以自由分享

除了基础 Copyleft 的做法以外，目前我还遵守以下做法：

- 不要求、建议或引导捐赠，且不在软件介绍网页、视频、软件官网或软件内加入「捐赠」相关内容

- 不在软件介绍网页、视频、软件官网或软件内加入 IAP 相关内容，包括付费课程，附加功能等

- 软件的不同发行只有发行方式上的差异（安装版/便携版），没有功能上的差异，即所有的用户都可以享受到软件所有的功能

- 不在软件介绍网页、视频、软件官网或软件内要求用户登录、注册、参加用户体验改进计划、启用遥测或以其他任何方式要求、建议或引导用户提供个人信息

- 不在软件介绍网页、视频、软件官网或软件内加入与软件或用户体验无关的信息，如天气、新闻等

- 在软件的启动页、关于页或其他明显的地方加入软件免费的提示，避免用户通过错误的途径获得本软件

# 反对

我反对如下做法。但世界永远比我们的认知魔幻得多，所以真的要有这样的事情发生了我也懒得管。

- 将本属于 Copyleft 的事物变成 Copyright 的

有许多软件本身即属于 Copyleft。如果你只是套了个壳就把软件变成 Copyright 的话，我反对。

- 将本身并不复杂的技术归属 Copyright

如果你真的研发了技术含量很高的算法，那我支持你将其归属 Copyright。

但如果你只是写了个批量脚本或是简单爬虫就想归属 Copyright 的话，我反对。

- 故意利用甚至制造技术相关的信息差并将「信息差」作为 Copyright

如果这项技术本身并不复杂，但你故意制造信息差并将其作为 Copyright，我反对。

最明显的一个例子就是，很多需求在 GitHub 上有很多优秀的实现，但一些人不向别人推荐 GitHub 上优秀的 Copyleft 实现，反而自己使用该 Copyleft 实现后利用信息差作为 Copyright 使用。

# 有关捐赠

我并非不接受捐赠。捐赠途径在页面底部，点按「赞赏」按钮即可显示捐赠二维码。但我希望你在捐赠之前看完此处列举的几点：

1. 我不缺钱。如果你的经济状况并不富裕的话，请务必不要捐赠。

1. 你的捐赠行为并不能直接满足你的需求。你可能想要一些特定的需求，但请注意，你的需求并不会因为你进行了捐赠而得到满足。在我认为你的需求合理的情况下，我会尽快完成你的需求，但这并不会因为你进行了捐赠而加快。
